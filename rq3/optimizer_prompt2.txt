Optimization rule (lexicographic):
- Candidate is strictly better if:
   - cand.two_qubit_gates < base.two_qubit_gates, OR
   - cand.two_qubit_gates == base.two_qubit_gates AND cand.volume < base.volume, OR
   - cand.two_qubit_gates == base.two_qubit_gates AND cand.volume == base.volume AND cand.depth < base.depth
- If all three metrics are equal or higher, it is NOT an improvement.

Note: two_qubit_gates counts ALL two-qubit entangling gates (CX, CZ, SWAP, etc.), not just CX.
Reducing any two-qubit gate type is equally valuable since all propagate errors between qubits.

## Optimization strategies
Think carefully about the mathematical structure of the stabilizer state before proposing candidates. Use these as a toolbox — not a fixed path. If one approach stalls after 3-4 attempts, switch to a different one.

- **Graph-state / LC-equivalent representation** — any stabilizer state is LC-equivalent to a graph state. A graph state on n qubits can be prepared with one CZ per edge plus local single-qubit gates. Reasoning about the underlying graph structure (which qubits need to be entangled) may reveal a much more compact circuit than the baseline.
- **Gaussian elimination on stabilizer generators** — treat the stabilizer generators as rows of a binary matrix over GF(2) and apply row operations to find a canonical form. Different orderings of generators yield different CNOT networks.
- **Qubit relabeling** — a permutation of qubit labels can expose cancellations or reduce the CNOT depth of the synthesis. Try heuristic relabelings and reason about which permutation minimizes entanglement overhead.
- **CNOT cancellation** — adjacent identical CX gates cancel (CX a b; CX a b = I). Triple patterns (CX a b; CX b a; CX a b) implement a SWAP and can sometimes be removed or replaced.
- **Gate commutation + peephole** — single-qubit gates commute through CX in predictable ways (X commutes through control, Z commutes through target, etc.). Use this to merge or cancel consecutive single-qubit gates on the same qubit.
- **Depth scheduling** — reorder commuting gates into parallel layers to reduce depth without changing gate count.
