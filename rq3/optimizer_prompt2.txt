Optimization rule (lexicographic):
- Candidate is strictly better if:
   - cand.two_qubit_gates < base.two_qubit_gates, OR
   - cand.two_qubit_gates == base.two_qubit_gates AND cand.volume < base.volume, OR
   - cand.two_qubit_gates == base.two_qubit_gates AND cand.volume == base.volume AND cand.depth < base.depth
- If all three metrics are equal or higher, it is NOT an improvement.

Note: two_qubit_gates counts ALL two-qubit entangling gates (CX, CZ, SWAP, etc.), not just CX.
Reducing any two-qubit gate type is equally valuable since all propagate errors between qubits.

## Strategies to try (in roughly this priority order)
Do NOT just name a strategy — actually implement it in a script or as a concrete circuit.

1. **Stim resynthesis** — extract tableau from baseline, call `tab.to_circuit(method="elimination")`
   and `tab.to_circuit(method="graph_state")`. Submit whichever has fewer two-qubit gates.
2. **Qubit permutation + resynthesis** — try several qubit relabelings before extracting the tableau,
   resynthesize, map qubits back. Often finds shorter circuits.
3. **Stabilizer generator reordering** — reorder the rows of the tableau before synthesis.
   Different row orderings yield different CNOT networks via Gaussian elimination.
4. **CNOT cancellation passes** — scan the circuit for adjacent inverse pairs (CX a b, CX a b → identity)
   and triple patterns (CX a b, CX b a, CX a b → SWAP). Apply cancellations then recheck validity.
5. **Gate commutation + peephole** — commute single-qubit gates through CX, merge consecutive
   single-qubit gates on the same qubit, eliminate identity gates.
6. **Depth scheduling** — reorder commuting gates into parallel layers to reduce depth without
   changing gate count.

Mix strategies: e.g., resynthesize first, then apply cancellation passes on the result.
If one strategy stalls after 3-4 attempts, switch to a different one rather than repeating variants.
