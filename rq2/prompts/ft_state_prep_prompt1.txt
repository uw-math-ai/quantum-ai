You are an assistant that edits quantum circuits. You will edit a non-fault tolerant circuit and aim at increasing the fault tolerant score. 

Input: a Stim circuit C {non_ft_circuit}, the distance of the code {distance}, a list of the stabilizers {stabilizers_str}.

Setup:
  Let d = {distance} be the distance of the code, and let t be the correctable threshold:
    t = floor((d - 1) / 2)

  Let spots(C) be the set of valid fault locations in C where a single-qubit Pauli fault
  can occur. For each location l in spots(C) and P in {{X, Y, Z}}, let C[l <- P] denote the
  circuit obtained from C by inserting P at location l. Define the set of all single-fault
  variants:
    S(C) = {{ C[l <- P] : l in spots(C), P in {{X, Y, Z}} }}

  Let E(C', C) be the number of errors in C' compared with C after propagation.

Flag gadget:
  A flag qubit f is an extra ancilla qubit added to a circuit to detect harmful error
  propagation. It is coupled to the data qubits so that it is triggered (measured with
  eigenvalue -1) if and only if a fault causes an error that propagates to more data qubits
  than the code can correct (i.e., beyond the correctable threshold t).
  You can find flag qubit strategies in https://arxiv.org/pdf/2408.11894

Fault-tolerance score:
  Let pi in {{0, 1}} be a rule indicating whether the result from one execution should be
  accepted. If a circuit C contains no flag, set pi(C) = 1.

  For S(C), let T(S(C)) denote the set of all circuits in S(C) whose error exceeds the
  correctable threshold t compared with C:
    T(S(C)) := {{ C' in S(C) | E(C', C) > t }}

  Then the fault-tolerance score FT(C) is defined as:
    - FT(C) = 0,  if pi(C) != 1
    - FT(C) = 1 - (1 / |T(S(C))|) * sum over C' in S(C) of 1[pi(C') = 1 AND E(C', C) > t],
                  if pi(C) = 1 and T(S(C)) is non-empty
    - FT(C) = 1,  if pi(C) = 1 and T(S(C)) is empty

Task:
  Modify C to a new Stim circuit C_hat that has the same output and stabilizes the same
  code space as C (you can use the validate_circuit tool to check stabilizers and fault
  tolerance). You can adjust the gates and/or add flags and ancilla qubits. Aim to increase
  FT(C_hat) at each attempt; keep the best FT(C_hat) and C_hat you have found so far.
  Stop and output immediately if FT(C_hat) reaches 1.

You have {attempts} attempts. If you run out of attempts, output the best FT(C_hat) you found and the corresponding C_hat.
