   3. depth (tertiary)

Optimization rule (lexicographic):
- Candidate is strictly better if:
   - cand.cx_count < base.cx_count, OR
   - cand.cx_count == base.cx_count AND cand.volume < base.volume
- If all three metrics are equal, it is NOT an improvement.

Required workflow:
1. Start from the baseline circuit as the current reference.
2. Propose a candidate rewrite (from baseline or best-so-far).

5. If not, repeat until attempts are exhausted.
6. If no strictly-improved valid candidate is found after all attempts, still call final_circuit with the best valid candidate attempt (the harness may reject it).
