Task: Generate a valid stabilizer circuit that preserves all given stabilizers and is fault-tolerant against single-qubit Pauli errors.
PRIMARY GOAL: build a circuit that preserves all stabilizers
SECONDARY GOAL: maximize ft_score
TERTIARY GOAL: minimize the number of gates in the circuit

Target Stabilizer State:
{stabilizers_str}

Circuit Constraints:
- Act on exactly {qubits_count} data qubits, indexed 0..{max_data_qubit}, starting from |0⟩^{{⊗ {qubits_count}}}.
- May use {num_flag_qubits} flag qubit(s), indexed {num_data_qubits}..{max_flag_qubit}, for error detection.
- The final quantum state must be a +1 eigenstate of every provided stabilizer generator.
- Produce a valid Stim circuit with at most {max_gates} gates.

You will be given a partial circuit, choose [the next gate, complete the circuit, OR next 10 steps]
Available Gates:
- Single-qubit gates: {one_qubit_gates}
  Apply by calling apply_one_qubit_gate with gate name and target qubit index.
- Two-qubit gates: {two_qubit_gates}
  Apply by calling apply_two_qubit_gate with gate name and control/target qubit indices.
- Standard basis measurements of flag qubits: call measure to terminate the circuit.

Objective and Reward Function:
You will be evaluated using an RL reward function with the following structure:
- Each gate adds a step penalty of {step_penalty}
- Each step provides incremental reward for stabilizer preservation (proportion of stabilizers preserved × {stabs_preserved_score_scale})
- Each step provides incremental reward for fault tolerance score (ft_score × {ft_score_scale})
- On termination: {success_reward} reward if all stabilizers are preserved AND circuit is fault-tolerant; {failure_penalty} penalty otherwise
- Any invalid action (control=target, exceeding max gates, etc.) incurs {failure_penalty} penalty and terminates immediately

Fault Tolerance Definition:
Fault tolerance is quantified using the ft_score metric, computed by injecting single-qubit Pauli faults (X, Y, Z) at each gate location and tracking their propagation through the circuit:

  ft_score = 1 - sum(weight(f)^p for undetected faults f with weight(f) > ⌊(d-1)/2⌋)
                 / sum(weight(f)^p for faults f with weight(f) > ⌊(d-1)/2⌋)

where:
  - weight(f) is the number of data qubits with errors after fault f propagates through the circuit
  - undetected means the fault causes no X error on any flag qubit
  - d is the code distance ({code_distance})
  - p is the power parameter ({ft_score_p})
  
An ft_score of 1.0 indicates perfect fault tolerance: all faults that propagate to multiple data qubits are detected by flag qubits. Lower scores indicate presence of undetected faults with high data-qubit weight. A circuit is fault-tolerant (and eligible for the success reward) when ft_score = 1.0.

Success Criteria:
- All {target_stab_count} stabilizer generators are preserved by the final circuit.
- Circuit passes fault-tolerance check.
- Circuit uses ≤ {max_gates} gates.
